# x-bookmark-deep-research

ブックマークを、ユーザー自身が残した未整理の関心ログとして扱う。単なる一覧化ではなく、俯瞰、深掘り候補の提示、ユーザー選択、追加調査、示唆レポート作成までを 1 つの skill で進める。

## いつ呼ぶか

- X のブックマーク済み投稿を一通り見て、深掘りするテーマを見つけたいとき。
- ブックマーク群から、関心の偏り、反復論点、未検証仮説、外れ値を見つけたいとき。
- ユーザーに複数の深掘り方向を提示し、選ばれた方向性でレポートまで作りたいとき。

呼ばないでよいケース:

- キーワード検索から通常の X 調査を始める → `x-search-plan`
- 既知の検索条件で一次取得だけ行う → `x-search`
- 取得済み JSON を事実レポートにするだけ → `x-search-report`
- 既存ファクトレポートから考察だけ作る → `x-search-insight`

## 成果物

- ブックマーク取得結果 JSON。
- ブックマーク俯瞰メモ: 関心クラスタ、代表投稿、薄い領域、外れ値。
- ユーザーに提示する深掘り候補 3〜5 個。
- 選択テーマに対する追加調査ログ。
- 最終の示唆レポート Markdown。固定テンプレートには縛られず、読者に最も示唆が伝わる構成を選ぶ。

## 手順

1. **ブックマーク取得条件を決める**
   - 既定は直近ブックマーク `--limit 100`。
   - ユーザーが「全部」「広く」「過去分も」と言った場合は `--all --max-pages 5` など上限付きで始める。
   - folder 指定がある場合は `--folder-id` を使う。
   - thread 文脈が重要そうなら `--include-parent` / `--thread-meta` を足す。最初から過剰な thread 展開はしない。

2. **`scripts/bookmarks.py` で取得する**
   - secret は aachat env provider から注入された `AUTH_TOKEN` / `CT0` を環境変数で受ける。値はログ・レポート・debug 出力に書かない。
   - 実行例:

   ```bash
   python3 .agents/skills/x-bookmark-deep-research/scripts/bookmarks.py \
     --limit 100 \
     --format json
   ```

   - 全件方向の例:

   ```bash
   python3 .agents/skills/x-bookmark-deep-research/scripts/bookmarks.py \
     --all \
     --max-pages 5 \
     --include-parent \
     --thread-meta \
     --format json
   ```

3. **ブックマークを俯瞰する**
   - 投稿本文、投稿者、指標、URL、thread metadata を読み、内容で仮クラスタリングする。
   - クラスタ数は 5〜10 を目安にする。少ない場合は無理に分けない。
   - クラスタ名は描写型にする。例: 「AI エージェントの実装知見」「創業者の意思決定」「研究論文の読み方」。
   - 「優良」「狙い目」「勝ち筋」など評価・施策語をクラスタ名に入れない。
   - 各クラスタには代表投稿 ID / URL を 1 件以上付ける。

4. **深掘り候補を提示する**
   - ユーザーに 3〜5 個の選択肢を出し、ここで必ず選択を待つ。
   - 各選択肢には次を含める:
     - 深掘りテーマ
     - ブックマーク上の根拠
     - なぜ深掘り価値があるか
     - 追加で見るべき対象
     - 期待できるレポートの形
   - 選択肢は互いに重複しすぎないようにする。
   - ユーザーが複数選んだ場合は、1 本で扱えるなら統合し、広すぎるなら分割を提案する。

5. **選択テーマを深掘りする**
   - 選ばれた代表投稿は `x-search expand` で thread / replies を確認する。
   - 関連語、著者、引用元、反証語が必要なら `x-search search` / `x-search account` を使う。
   - 追加調査では支持材料だけでなく、反証、別解釈、サンプル偏りも探す。
   - 深掘り対象を増やしすぎない。最終レポートに効く投稿・会話・検索だけに絞る。

6. **示唆レポートを書く**
   - テンプレートを固定せず、テーマに応じて最も伝わる構成にする。
   - ただし最低限、以下は含める:
     - 何を見たか
     - 重要な観察
     - そこから読める示唆
     - 根拠投稿 / 根拠検索
     - 確信度と理由
     - 反証・注意点
     - 次に調べると価値がある論点
   - 各主張にはブックマーク投稿 ID、追加取得投稿 ID、または検索ログ ID を付ける。
   - ファクトと解釈は混ぜず、解釈には必ず「根拠」「確信度」「弱点」を添える。

## 深掘り候補の出し方

良い候補は、次のいずれかを満たす。

- 複数クラスタにまたがって同じ問いが反復している。
- 単一クラスタだが、代表投稿の密度が高く、追加調査で構造化できそう。
- ブックマーク群の中で外れ値だが、ユーザーの新しい関心の兆しに見える。
- 強い主張があるが、反証や条件がまだ見えていない。
- 既存の `x-search` で thread / replies / related query を深掘りできる。

## レポート品質ルール

- 「ブックマークされた」こと自体は関心シグナルだが、同意や支持とは限らない。
- 単一投稿から市場全体、業界全体、X 全体へ一般化しない。
- エンゲージメント数は重要度ではなく、会話量や拡散の補助指標として扱う。
- 投稿者の権威やフォロワー数だけで主張の確信度を上げない。
- 皮肉、引用 RT、反論文脈、thread の途中投稿は誤読しやすいので注意点に書く。

## やらないこと

- ユーザー選択なしで、勝手に 1 つの深掘りテーマへ進まない。
- ブックマークしていない投稿を、根拠なしにユーザーの関心として扱わない。
- secret、token、cookie、raw credential、内部ログを出力しない。
- 投稿文、広告文、クリエイティブ案を主成果物にしない。
- ブックマークの削除、投稿、返信、DM 送信など状態変更を行わない。

## 参考

- `scripts/bookmarks.py`: bird `bookmarks` を呼び、既存 `x-search` 互換に近い item JSON を返す。
- 隣の `x-search` skill: 選択テーマの thread / replies / 関連検索に使う。
- 隣の `x-search-report` skill: 追加取得結果を事実整理したい場合に使う。
- 隣の `x-search-insight` skill: 追加ファクトレポートから考察を分離したい場合に使う。
